## 1. 문제 개요

> **Level Goal**
> 
> The password for the next level can be retrieved by submitting the password of the current level to **port 30000 on localhost**.

이번 레벨은 네트워크 통신의 기초를 다룹니다. 현재 레벨의 비밀번호를 가지고 로컬 서버의 특정 포트(30000번)에 접속해서 "나 비밀번호 알아요"라고 말해주면, 서버가 다음 레벨 비밀번호를 알려주는 구조임을 파악했습니다.

---

## 2. 사용 명령어

| 명령어 | 설명 |
|--------|------|
| `nc [호스트] [포트]` | Netcat. 네트워크 연결을 읽고 쓰는 만능 도구 ("네트워크의 스위스 아미 나이프") |
| `telnet` | (오래된 방식) 원격 접속 도구. 평문 전송이라 보안에 취약하지만 간단한 테스트용으로 쓰임 |

---

## 3. 풀이 과정

이전 레벨에서 얻은 키를 이용해 `bandit14`로 접속한 상태여야 합니다.

```bash
# bandit13에서 접속 시
ssh -i sshkey.private bandit14@localhost -p 2220
```

### 1. 현재 비밀번호 확인
먼저 이번 레벨(`bandit14`)의 비밀번호가 필요합니다.
`/etc/bandit_pass/bandit14` 파일을 읽어서 복사해둡니다.

   비밀번호: `4YjL3GvafeR3b2hTUtbT3bTGo22a4D22`

### 2. 포트 접속 (`nc`)
`nc` (Netcat) 명령어로 `localhost`의 `30000` 포트에 접속합니다.

```bash
nc localhost 30000
```
아무런 메시지가 안 뜨고 커서만 깜빡이는데, 연결된 상태입니다.

### 3. 비밀번호 전송
방금 복사한 `bandit14`의 비밀번호를 붙여넣고 엔터를 칩니다.

```bash
4YjL3GvafeR3b2hTUtbT3bTGo22a4D22
```

### 4. 결과 확인
서버가 비밀번호를 확인하고 맞으면 `Correct!` 메시지와 함께 다음 레벨 비밀번호를 보내줍니다.

### 더 멋진 풀이 (One-liner)

`echo`와 파이프(`|`)를 사용하면 복사/붙여넣기 없이 한 줄로 해결할 수 있습니다.

```bash
echo "4YjL3GvafeR3b2hTUtbT3bTGo22a4D22" | nc localhost 30000
```
또는 파일에서 바로 읽어서 보낼 수도 있습니다:
```bash
cat /etc/bandit_pass/bandit14 | nc localhost 30000
```

## 5. 배운 점

1. **포트(Port) 개념**: IP 주소가 "집 주소"라면, 포트는 "방 번호"와 같습니다. 하나의 서버 안에서 웹(80), SSH(22), 메일(25) 등 여러 서비스가 서로 다른 포트를 사용하여 동시에 실행됩니다.
2. **Netcat (`nc`)**: 해킹이나 보안 점검 시 가장 많이 쓰이는 도구 중 하나입니다. 포트 스캔, 데이터 전송, 백도어 생성 등 활용도가 무궁무진합니다.

---

## 6. 보안 관점

- **평문 통신 위험**: `nc`나 `telnet`은 기본적으로 데이터를 암호화하지 않고 그대로 보냅니다. 만약 누군가 네트워크를 감청(Sniffing)하고 있다면, 우리가 방금 보낸 비밀번호를 그대로 훔쳐볼 수 있습니다. 
- 따라서 실무에서는 데이터를 전송할 때 반드시 **SSL/TLS (HTTPS, SSH 등)** 로 암호화된 채널을 사용해야 합니다.
