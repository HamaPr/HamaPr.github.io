## 1. 문제 개요

> **Level Goal**
> 
> This is the last level of the bandit wargame. The password for this level is not for the next level, but for the next wargame: **leviathan**.

드디어 Bandit 워게임의 마지막 단계입니다. 이번 레벨은 문제를 푼다기보다, 지금까지의 여정을 마무리하고 다음 단계인 **Leviathan** 워게임으로 넘어가는 관문이었습니다. Bandit을 통해 무엇을 배웠는지 정리해 보았습니다.

---

## 2. 사용 명령어

| 명령어 | 설명 |
|--------|------|
| `ssh` | 다른 서버(Leviathan)로 접속 |

## 3. 풀이 과정

`bandit33` 계정으로 로그인합니다.

```bash
ssh bandit33@bandit.labs.overthewire.org -p 2220
```

### 1. 홈 디렉토리 확인
아무 파일도 없습니다. 숨겨진 파일도 없습니다.

### 2. README 확인 (Level 33)
`README.txt`가 어딘가에는 있었던 기억이 나서 찾아보거나, 문제 설명을 다시 보니 "At this moment, level 34 does not exist yet."이라고 합니다. 대신 `bandit33`의 비밀번호가 사실상 마지막이라고 합니다.

```bash
bandit33@bandit:~$ cat README.txt
Congratulations on solving the last level of this wargame!
At this moment, there are no more levels to play in this game.
...
The password for the user bandit34 is the same as for the user bandit33.
The password for the next wargame, leviathan, is in the file /etc/bandit_pass/bandit34.
```

2. **비밀번호 획득**: 안내에 따라 `/etc/bandit_pass/bandit34` 파일을 읽습니다.

```bash
bandit33@bandit:~$ cat /etc/bandit_pass/bandit34
c9c31... (이전 레벨과 동일)
```
Bandit 서버에서는 Leviathan 비밀번호를 바로 주지 않고, **이 비밀번호를 가지고 Leviathan 게임을 시작해라**라고 안내합니다.

### Leviathan 접속 방법
Bandit과 똑같습니다. 주소만 `leviathan.labs.overthewire.org`로 바꾸면 됩니다.

```bash
ssh leviathan0@leviathan.labs.overthewire.org -p 2223
```
(비밀번호는 `bandit33`의 비밀번호를 그대로 사용하면 됩니다.)

## 4. 결과

**Bandit 졸업**
 
 Level 0부터 34까지 오면서 리눅스 시스템 해킹의 기초를 탄탄히 다질 수 있었습니다.

### Bandit 보안 개념 요약

| 주제 | 관련 레벨 | 핵심 내용 |
|---|---|---|
| **기본 명령어** | 0~4 | `ls`, `cat`, `file`, 숨김 파일, 파일명 공백 처리 |
| **권한 관리** | 5~9, 19~20 | 파일 소유권, 그룹, **SetUID**를 이용한 권한 상승 |

## 5. 소감

1. **리눅스와 친해지기**: 검은 바탕에 흰 글씨(터미널)가 더 이상 무섭지 않게 되었습니다. 모든 해킹의 기본은 운영체제(OS)에 대한 깊은 이해에서 시작된다는 것을 깨달았습니다.
2. **Everything is a File**: 리눅스에서는 설정, 명령어, 데이터, 심지어 하드웨어 장치까지 모든 것이 파일로 다뤄진다는 철학을 몸소 체험했습니다.
3. **끈기**: 34개의 문제를 풀면서 막히는 순간도 있었겠지만, 검색하고 시도하며 끝까지 해결해낸 경험이 가장 큰 자산이 될 것입니다.

---

## 6. 보안 관점

- **White Hat Hacker**: 지금까지 시스템의 취약점을 찾아내고 공격하는 법을 배웠습니다. 하지만 이 기술은 범죄를 저지르기 위함이 아니라, **시스템을 더 안전하게 만들기 위함**임을 명심해야 합니다.
- **Security Mindset**:
    - **개발자의 실수 확인**: 백업 파일, Git 히스토리 등에 민감한 정보가 남아있는지 점검
    - **입력값 검증 미비점 탐색**: 사용자 입력을 검증 없이 실행하여 커맨드 인젝션이 가능한지 확인
    - **과도한 권한 설정 점검**: SetUID나 777 권한이 불필요하게 부여되어 있는지 감사(Audit)
    이런 의심(Critical Thinking)을 멈추지 않는 것이 보안 전문가의 자세입니다.
