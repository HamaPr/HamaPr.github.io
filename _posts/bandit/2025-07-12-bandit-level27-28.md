## 1. 문제 개요

> **Level Goal**
> 
> There is a **git repository** at `ssh://bandit27-git@localhost/home/bandit27-git/repo`. The password for the user `bandit27-git` is the same as for the user `bandit27`.
> Clone the repository and find the password for the next level.

이제부터는 **Git(깃)** 시리즈입니다. Git은 현대 개발자에게 없어서는 안 될 버전 관리 시스템입니다. 이번 레벨에서는 원격 저장소에 있는 코드를 내 컴퓨터(여기서는 `/tmp` 디렉토리)로 가져오는 **Clone(복제)** 작업을 수행해야 했습니다.

---

## 2. 사용 명령어

| 명령어 | 설명 |
|--------|------|
| `git clone [URL]` | 원격 저장소(Repository)를 로컬로 복제하여 다운로드 |
| `mkdir [디렉토리]` | 새 디렉토리 생성 |

---

## 3. 풀이 과정

`bandit27` 계정으로 로그인합니다.

```bash
ssh bandit27@bandit.labs.overthewire.org -p 2220
```

### 1. 작업 공간 만들기
우리는 홈 디렉토리에 쓰기 권한이 없습니다. 따라서 `/tmp` 디렉토리 아래에 나만의 작업 공간을 만들어야 합니다.

```bash
bandit27@bandit:~$ mkdir /tmp/mygit27
bandit27@bandit:~$ cd /tmp/mygit27
```

### 2. Git Clone
문제에서 주어진 주소(`ssh://...`)로부터 저장소를 복제합니다.

```bash
bandit27@bandit:/tmp/mygit27$ git clone ssh://bandit27-git@localhost/home/bandit27-git/repo
Cloning into 'repo'...
...
bandit27-git@localhost's password:
```
비밀번호를 물어보면 현재 레벨(`bandit27`)의 비밀번호를 입력합니다.

### 3. 내용 확인
복제가 완료되면 `repo`라는 폴더가 생깁니다. 들어가서 내용을 봅니다.

```bash
bandit27@bandit:/tmp/mygit27$ cd repo
bandit27@bandit:/tmp/mygit27/repo$ ls
README
bandit27@bandit:/tmp/mygit27/repo$ cat README
The password for the next level is: 0efRyCNt...
```
`README` 파일에 친절하게 다음 비밀번호가 적혀 있습니다.

## 5. 배운 점

1. **Git의 기본**: 소스 코드를 내 컴퓨터로 가져오는 `clone` 명령어를 익혔습니다.
2. **쓰기 권한**: 리눅스 공용 서버에서 작업할 때는 `/tmp` 디렉토리가 유용한 작업 공간이 된다는 점을 다시 상기했습니다.

---

## 6. 보안 관점

- **.git 디렉토리 유출**: 웹사이트를 배포할 때 실수로 `.git` 폴더까지 같이 올려버리면 심각한 보안 사고가 발생합니다. 누구든지 `git clone http://사이트/.git` 등으로 소스코드 전체와 변경 이력(비밀번호가 포함된 과거 코드 등)을 다운로드할 수 있기 때문입니다.
