## 1. 문제 개요

> **Level Goal**
> 
> The password for the next level is stored in the file `data.txt`, where all lowercase (a-z) and uppercase (A-Z) letters have been **rotated by 13 positions**.

비밀번호가 **ROT13**이라는 고전적인 기법으로 변환(회전)되어 있었습니다. 알파벳 순서에서 13번째 뒤의 문자로 단순 치환하는 방식입니다.

---

## 2. 사용 명령어

| 명령어 | 설명 |
|--------|------|
| `tr [SET1] [SET2]` | 문자열 변환(Translate). SET1의 문자들을 SET2의 문자로 1:1 치환합니다. |
| `[a-z]` | 소문자 전체 (a부터 z까지) |
| `[A-Z]` | 대문자 전체 (A부터 Z까지) |

---

## 3. 풀이 과정

`bandit11` 계정으로 로그인합니다.

```bash
ssh bandit11@bandit.labs.overthewire.org -p 2220
```

### 1. 파일 내용 확인
`data.txt`를 보니 알파벳들이 뒤죽박죽 섞여 있습니다.
문제에서 "모든 소문자(a-z)와 대문자(A-Z)가 13글자씩 밀렸다"고 힌트를 줬습니다. 이것이 바로 **ROT13** 암호입니다.

```bash
bandit11@bandit:~$ cat data.txt
Gur cnffjbeq vf...
```

알아볼 수 없는 문장이지만, 공백과 문장 구조는 유지되어 있습니다. 단순 치환 암호임을 짐작할 수 있습니다.

### 2. 복호화
`tr` 명령어로 문자 치환을 수행합니다.
- `a-z` (소문자)는 `n-za-m` (n부터 z까지, 그 뒤에 a부터 m까지)으로 바꿉니다.
- `A-Z` (대문자)는 `N-ZA-M`으로 바꿉니다.

알파벳은 총 26글자이므로, 13글자를 밀면 정확히 반바퀴를 돌게 됩니다.

```bash
bandit11@bandit:~$ cat data.txt | tr 'A-Za-z' 'N-ZA-Mn-za-m'
The password is Tf5W2ETppYgFPKJ2e55b5g75sKAlAWk7
```

명령어 해석: "A부터 Z는 N부터 시작하는 순서로 바꾸고, a부터 z는 n부터 시작하는 순서로 바꿔라"

## 5. 배운 점

1. **`tr` 명령어 활용**: 불필요한 문자 삭제(`-d`), 소문자를 대문자로 변환(`tr 'a-z' 'A-Z'`) 등 텍스트 처리에 유용합니다.
2. **시저 암호 (Caesar Cipher)**: ROT13은 시저 암호의 일종으로, 단순히 글자 위치만 바꾸는 방식이라 보안성은 전혀 없습니다.

---

## 6. 보안 관점

- **취약한 암호 알고리즘**: ROT13이나 단순 치환 방식은 현대 컴퓨터로는 순식간에 해독 가능합니다. 비밀번호 보호를 위해선 AES 같은 **강력한 암호화 알고리즘**이나 SHA-256 같은 **해시 함수**를 사용해야 합니다.
- **난독화(Obfuscation) vs 암호화(Encryption)**: 코드를 읽기 어렵게 만드는 것(난독화)과 수학적으로 정보를 보호하는 것(암호화)은 다릅니다. ROT13은 일종의 난독화에 가깝습니다.
