## 1. 문제 개요

> **Level Goal**
> 
> The password for the next level is stored in the file `data.txt` in one of the few **human-readable strings** (사람이 읽을 수 있는 문자열), preceded by several '=' characters.

`data.txt`는 텍스트 파일이 아니라 바이너리 데이터가 섞인 파일입니다. `cat`으로 열면 터미널이 깨질 수 있습니다. 이런 파일에서 의미 있는 텍스트 정보만 추출해야 했습니다. 힌트는 비밀번호 앞에 `=` 문자가 여러 개 있다는 점입니다.

---

## 2. 사용 명령어

| 명령어 | 설명 |
|--------|------|
| `strings [파일]` | 바이너리 파일 내에서 **출력 가능한(Printable) 문자열**만 추출하여 보여줍니다. |
| `grep [패턴]` | 지정한 패턴을 검색합니다. |

---

## 3. 풀이 과정

`bandit9` 계정으로 로그인합니다.

```bash
ssh bandit9@bandit.labs.overthewire.org -p 2220
```

### 1. 파일 확인
`data.txt`는 "human-readable"한 문자열이 섞여 있는 바이너리 파일입니다.
`cat`으로 그냥 열어보면 깨진 글자들이 화면을 뒤덮고 삑삑 소리가 날 것입니다.

```bash
bandit9@bandit:~$ file data.txt
data.txt: data
```

`data`라고 나옵니다. 이는 특정 포맷이 알려지지 않은 바이너리 파일이라는 뜻입니다.

### 2. 문자열 추출 (strings)
이럴 때 쓰는 명령어가 `strings`입니다. 바이너리 파일 안에서 "사람이 읽을 수 있는(printable)" 문자열만 골라서 보여줍니다.

```bash
strings data.txt
```
이렇게 하면 화면에 읽을 수 있는 단어들이 주르륵 나옵니다.

### 3. 조건 필터링 (grep)
문제에서 비밀번호는 **`=`(Equal sign) 여러 개**로 시작한다고 했습니다.
`grep`으로 `=` 표시가 있는 줄을 찾습니다.

```bash
strings data.txt | grep "=="
```
```
========== the
========== password
========== is
========== FGUW5ilLVJrxX9kMYMmlN4MgbpfMiqey
```

`strings`가 바이너리 노이즈를 다 걸러내고 텍스트만 보여주었기 때문에 쉽게 비밀번호를 찾을 수 있었습니다.

## 5. 배운 점

1. **`strings` 명령어**: 해커나 리버스 엔지니어링 분석가가 가장 즐겨 쓰는 도구 중 하나입니다. 실행 파일(.exe, ELF)이나 이미지 파일 등 겉보기에 텍스트가 아닌 파일 안에 숨겨진 정보를 찾을 때 매우 유용합니다.
2. **바이너리 vs 텍스트**: 컴퓨터 파일은 크게 사람이 읽을 수 있는 텍스트 파일과, 컴퓨터가 해석하는 바이너리 파일로 나뉜다는 점을 이해했습니다.

---

## 6. 보안 관점

- **하드코딩된 정보 유출**: 개발자들이 종종 프로그램 소스 코드나 컴파일된 바이너리 파일 안에 API Key, 비밀번호, 내부 서버 주소 등을 실수로 남겨둡니다. ("Hardcoding")
- **리버스 엔지니어링 기초**: 공격자는 앱을 다운로드한 후 가장 먼저 `strings`를 돌려봅니다. 만약 중요한 정보가 평문으로 들어있다면 즉시 유출됩니다. 따라서 민감한 정보는 절대 코드에 직접 적지 말고, 환경 변수나 보안 저장소를 사용해야 합니다.
