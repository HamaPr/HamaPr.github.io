## 1. Java RMI 개요

**Java RMI (Remote Method Invocation)**는 자바 프로그램이 네트워크 상의 다른 자바 가상 머신(JVM)에 있는 객체의 메소드를 호출할 수 있게 해주는 기술입니다. 분산 애플리케이션을 쉽게 개발할 수 있도록 도와주지만, 보안 설정이 미흡할 경우 심각한 위협이 될 수 있습니다.

RMI는 기본적으로 1099번 포트를 사용하며, 객체를 직렬화(Serialization)하여 네트워크로 전송하고, 수신 측에서 역직렬화(Deserialization)하여 사용합니다.

## 2. 취약점 분석

이번 시나리오에서는 **Java RMI** 서비스가 외부에서 접근 가능하도록 설정되어 있어 발생할 수 있는 문제를 다룹니다.

### 2.1. RMI 레지스트리 접근
RMI 레지스트리가 외부에 노출되어 있으면, 공격자는 바인딩된 객체 목록을 조회하거나(`list`), 악의적인 객체를 바인딩(`bind`/`rebind`)할 수 있습니다. 이를 통해 애플리케이션의 흐름을 조작하거나 정보를 탈취할 수 있습니다.

### 2.2. 역직렬화(Deserialization) 취약점
가장 치명적인 공격은 **역직렬화 취약점**입니다. 자바의 직렬화 메커니즘은 신뢰할 수 없는 데이터를 역직렬화할 때, 객체 내부의 코드가 자동으로 실행될 수 있는 구조적 문제를 가지고 있습니다.

공격자는 `ysoserial`과 같은 도구를 사용하여 악성 페이로드가 포함된 직렬화된 객체를 생성하고, 이를 RMI 서비스로 전송합니다. 서버가 이 객체를 역직렬화하는 순간, 공격자가 의도한 시스템 명령어가 실행(RCE)됩니다.

## 4. 보안 대책

1.  **네트워크 격리**: RMI 포트(기본 1099)는 외부에서 접근할 수 없도록 방화벽으로 차단하고, 내부 네트워크 또는 로컬호스트에서만 접근 가능하도록 제한합니다.
2.  **SSL/TLS 적용**: RMI 통신에 SSL/TLS를 적용하여 데이터 전송 구간을 암호화하고, 클라이언트 인증을 수행합니다.
3.  **Security Manager 사용**: Java Security Manager를 활성화하고 엄격한 정책 파일을 적용하여, RMI를 통해 로드되는 클래스의 권한을 제한합니다.
4.  **최신 라이브러리 사용**: 역직렬화 취약점이 패치된 최신 버전의 JDK와 라이브러리를 사용합니다.
